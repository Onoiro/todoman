# TodoMan - приложение для управления списком дел
Разработано в рамках итогового проекта курса "Первые шаги в JavaScript" от Selectel

## Особенности

- **Добавление и удаление задач** — быстро создавайте новые задачи и удаляйте выполненные
- **Отметить как выполнено** — отслеживайте прогресс выполнения задач
- **Фильтрация** — просматривайте все задачи, только активные или завершённые
- **Локальное хранилище** — все задачи сохраняются в браузере автоматически
- **Адаптивный дизайн** — работает на мобильных телефонах, планшетах и десктопе
- **Touch-события** — оптимизирован для сенсорных экранов

## Быстрый старт

### Требования
- Современный браузер (Chrome, Firefox, Safari, Edge)
- Нет необходимости в сервере или установке зависимостей

### Установка

1. Клонируйте репозиторий:
```bash
git clone https://github.com/Onoiro/todoman.git
cd todoman
```

2. Откройте файл `index.html` в браузере:
```bash
# Вариант 1: Просто откройте файл в браузере
xdg-open index.html  # Linux
open index.html      # macOS
start index.html     # Windows

# Вариант 2: Запустите локальный сервер (опционально)
python3 -m http.server 8000
# Откройте http://localhost:8000 в браузере
```

## Как пользоваться

### Добавление задачи
1. Введите текст задачи в поле ввода
2. Нажмите кнопку «Добавить» или нажмите Enter
3. **Ограничение:** максимум 100 символов на задачу

### Управление задачами
- **Отметить как выполнено** — кликните на чекбокс слева от задачи
- **Удалить задачу** — нажмите кнопку «×» справа от задачи

### Фильтрация задач
- **Все** — показываются все задачи
- **Активные** — показываются только незавершённые задачи
- **Завершённые** — показываются только выполненные задачи

### Поиск задач
- Введите текст в поле поиска для фильтрации задач по содержимому
- Поиск работает вместе с фильтрами по статусу

### Сохранение данных
- Все задачи автоматически сохраняются в localStorage браузера
- Данные сохраняются между сессиями и при закрытии браузера

```
todoman/
├── index.html    # HTML-разметка приложения
├── style.css     # Стили и адаптивный дизайн
├── app.js        # Логика приложения
```

### Описание файлов

- **index.html** — содержит структуру интерфейса: форму добавления задач, поле поиска, кнопки фильтров и список задач
- **style.css** — стили с mobile-first подходом, адаптивные брейкпоинты, touch-оптимизация
- **app.js** — основная логика: добавление, удаление, фильтрация, поиск, localStorage

## Технологический стек

- **HTML5** — семантическая разметка
- **CSS3** — стили, Flexbox, медиа-запросы, CSS-переменные
- **JavaScript (ES6+)** — логика приложения, работа с localStorage, event handling
- **localStorage API** — клиентское хранение данных

## Адаптивность

Приложение использует **mobile-first** подход с тремя брейкпоинтами:

| Устройство | Ширина экрана | Особенности |
|------------|---------------|-------------|
| Мобильные | < 768px | Полноширинные элементы, крупные кнопки (44×44px), вертикальная компоновка |
| Планшеты | 768px – 1023px | Умеренные отступы, горизонтальное расположение элементов формы |
| Десктоп | ≥ 1024px | Максимальная ширина 600px, увеличенные отступы, hover-эффекты |

### Особенности адаптивного дизайна

- **Адаптивные кнопки:** минимальный размер 44×44px для удобного нажатия пальцем
- **Адаптивный текст:** базовый размер 14px на мобильных, увеличивается на десктопе
- **Адаптивные поля ввода:** 100% ширины на мобильных, ограничена на десктопе
- **Touch-события:** мгновенный отклик на сенсорных экранах (touchstart/touchend)
- **Hover-эффекты:** применяются только на устройствах с поддержкой hover (десктоп)
- **Адаптивные отступы:** пропорционально изменяются с размером экрана
